# Simulator Control Panel

> Отдельный инструмент для управления виртуальным тепловозом в реальном времени.
> Данные идут через тот же pipeline что и реальная телеметрия — бэкенд не различает источник.

## Назначение

- Демо жюри: управление параметрами в реальном времени
- Тестирование: проверка алертов, HI, графиков
- Нагрузочное тестирование: 1700 виртуальных локомотивов
- Разработка: воспроизводимые сценарии

## Интеграция

```
Simulator Panel ──POST /ingest/telemetry──► Ingestion Service
                                                │
                (тот же JSON-формат              ▼
                 что и реальный тепловоз)     Kafka → Processor → Redis → WS Hub
                                                                    │
                                                              Dashboard (live)
```

Бэкенд не знает, пришли данные от реального тепловоза или от симулятора.
Формат пакета идентичен (см. docs/PARAMETERS.md).

## Доступ

- URL: `/simulator` (основной), `/simulator/loadtest` (highload)
- Роль: `admin` или `simulator`
- Не доступен машинисту/диспетчеру

## Три режима работы

### 1. Ручной режим
Каждый параметр управляется слайдером. Значения отправляются с выбранной частотой (1–10 Гц).

### 2. Авто-симулятор
Параметры колеблются автоматически вокруг базовых значений (шум ±2–5%).
Слайдеры задают «базовую точку», симулятор добавляет реалистичные колебания.

### 3. Сценарий
Готовый скрипт автоматически анимирует параметры по заданному паттерну.
Слайдеры двигаются сами — видно что происходит.

## Визуализация слайдеров

Каждый слайдер показывает зоны прямо на полоске:

```
Температура воды (выход):       текущее: 87
       0        71        91       115    150
  [серый|серый|ЗЕЛЁНЫЙ|=====|ЖЁЛТЫЙ|КРАСНЫЙ]
   crit   warn  НОРМА   ^87   warn   crit
```

Цвет числа справа меняется по зоне: зелёный / жёлтый / красный.

## Готовые сценарии

| Сценарий | Длительность | Что происходит | Ожидаемая реакция дашборда |
|---|---|---|---|
| **Перегрев воды** | 30 сек | water_temp_outlet: 87→100→116 | HI падает, жёлтый→красный, алерт, LCAS: "ТЕМПЕРАТУРА ВОДЫ КРИТИЧНА" |
| **Перегрев масла** | 30 сек | oil_temp_outlet: 79→90→98 | HI падает, алерт, LCAS: "ТЕМПЕРАТУРА МАСЛА КРИТИЧНА" |
| **Утечка воздуха** | 45 сек | main_reservoir: 8.8→7.5→6.5, компрессор вкл/выкл часто | HI падает, алерт пневмосистема |
| **Разгон-торможение** | 60 сек | speed: 0→80→100→60→0, температуры и расход коррелируют | Графики двигаются синхронно, красиво |
| **Тяжёлый участок** | 45 сек | Всё на верхней границе нормы | HI ~70, жёлтый, LCAS: "ВНИМАНИЕ" |
| **Каскадный отказ** | 90 сек | Масло→вода→земля→стоп | Лавина алертов, HI до 0, LCAS: серия голосовых |
| **Нормальная работа** | бесконечно | Стабильные значения ± 2% | HI 90+, зелёный, тишина |

## Корреляция параметров (авто-режим)

При изменении скорости остальные параметры реагируют с реалистичной задержкой:

```
Скорость растёт:
  → расход топлива растёт (мгновенно)
  → тяговый ток растёт (мгновенно)
  → температуры растут (задержка ~30 сек)
  → давление масла чуть снижается

Остановка:
  → скорость 0
  → температуры падают плавно (~60 сек до стабилизации)
  → компрессор периодически включается
  → расход топлива = idle (~30 л/ч)
```

## Маршруты

| Маршрут | Координаты от/до | Расстояние | Назначение |
|---|---|---|---|
| Астана — Караганда | 51.17,71.45 → 49.80,73.10 | ~230 км | основной демо |
| Алматы — Шу | 43.24,76.95 → 43.60,73.76 | ~300 км | альтернативный |
| Кольцевой (демо) | замкнутый контур | ~50 км | бесконечное демо |

Позиция пересчитывается каждую секунду:
```
distance_delta = speed_kmh / 3600   # км за 1 сек
position += distance_delta
lat, lon = interpolate(route_points, position)
```

Ограничения скорости на маршруте отображаются на карте симулятора и на карте диспетчера.

## Highload-тест

При запуске N виртуальных локомотивов:
- Каждый получает случайный маршрут, стартовую позицию и скорость (40–100 км/ч)
- Параметры — нормальные значения + шум ±3%
- ~5% локомотивов — в зоне "внимание" (для реалистичности)
- ~1% — в зоне "критично"

На карте диспетчера — россыпь движущихся точек.

## Демо-сценарий для жюри (3 минуты)

```
0:00  Дашборд спокоен. HI: 94. Тишина.
      (simulator: нормальная работа)

0:30  "Давайте устроим перегрев"
      (simulator: сценарий "Перегрев воды")
      → температура ползёт, HI падает, LCAS: "ВНИМАНИЕ. ТЕМПЕРАТУРА ВОДЫ"
      → дальше: "ТЕМПЕРАТУРА ВОДЫ КРИТИЧНА. СНЯТЬ НАГРУЗКУ"

1:00  "Стоп, возвращаем в норму"
      → алерт resolved, HI восстанавливается

1:30  "Replay — диспетчер перематывает последнюю минуту"
      → /dispatch/loco/:id/replay

2:00  "Нагрузочный тест — 1700 локомотивов"
      → /dispatch: 1700 точек на карте
      → /admin/system: RPS 1700, latency 48ms

2:30  "x10 пик — 17 000 RPS"
      → latency < 500мс, UI не тормозит, HPA скейлит pods

3:00  "Спасибо. Вопросы?"
```
