# User Flows — задание для дизайнера

> Проект: Цифровой двойник локомотива ТЭ33А
> Дашборд реального времени с индексом здоровья и телеметрией

---

## Общие требования

- **Две темы:** светлая + тёмная (переключатель в header)
- **Два формата:** 24" монитор (1920×1080) + ноутбук (1366×768)
- **Accessibility:** контрастные цвета, шрифт от 14px, читаемость с расстояния 1 м (для машиниста)
- **Цветовая система статусов:**
  - Зелёный — норма (HI 80–100)
  - Жёлтый — внимание (HI 50–79)
  - Красный — критично (HI 0–49)
- **Три роли:** Машинист (read-only), Диспетчер (read + actions), Администратор (настройки)

## Экран 1: Логин (`/login`)

**Назначение:** Авторизация пользователя и редирект по роли.

### Элементы
- Логотип / название системы: "Цифровой двойник ТЭ33А"
- Поле: Логин
- Поле: Пароль (с иконкой показать/скрыть)
- Кнопка: "Войти"
- Сообщение об ошибке: "Неверный логин или пароль"

### Поведение
- Успешный логин — редирект:
  - role=driver → `/cabin`
  - role=dispatcher → `/dispatch`
  - role=admin → `/admin`

### Описание виджетов

**Индекс здоровья (главный виджет):**
- Круговой gauge (arc/donut), крупный — ~200x200px
- Число в центре: значение 0–100 + буква A–E
- Цвет дуги: зелёный/жёлтый/красный по категории
- Подпись снизу: "Норма" / "Внимание" / "Критично"
- При клике — разворачивается панель top-5 факторов:
  ```
  Факторы снижения HI:
  1. Темп воды вых   94 C    -5.5
  2. Темп масла вых  88 C    -3.8
  3. Давление ТМ     4.6     -3.2
  (прогресс-бар вклада каждого фактора)
  ```

**Скорость:**
- Крупные цифры: 72 км/ч
- Gauge или спидометр (полукруг, шкала 0–120)
- Индикатор боксования: иконка колеса, при wheel_slip=true — красная + анимация

**4 панели параметров (Температуры / Давления / Электрика / Топливо):**
- Каждое значение — число + единица + цветовой фон по статусу
- Норма — нейтральный фон
- Warning — жёлтый фон, число жёлтое
- Critical — красный фон, число белое/мигающее
- Мини-спарклайн рядом с числом (опционально) — тренд за 30 сек

**Графики трендов:**
- Line chart, 2–3 параметра одновременно
- Автоскейлинг по Y
- Зоны на фоне: зелёная (норма), жёлтая (warning), красная (critical)
- Tooltip при наведении: точное значение + timestamp
- Переключатель периода: 1 мин / 5 мин / 15 мин

**Карта участка:**
- Упрощённая линия маршрута A — B
- Точка текущего положения
- Отметки ограничений скорости на участке

### Интерактивность (минимум)
- Клик на Индекс здоровья — раскрытие/сворачивание top-5 факторов
- Клик на графике — zoom/tooltip
- Переключатель периода графика (1/5/15 мин)
- Переключатель темы в header
- Всё остальное — только просмотр (read-only)

## Экран 3: Кабина — критический алерт (overlay поверх экрана 2)

### Состояние "Внимание" (warning)

Отдельного экрана нет — меняется вид виджетов на экране 2:
- Виджет параметра: фон — жёлтый
- Индекс здоровья: дуга — жёлтая
- Алерт-карточка появляется в блоке алертов:
  ```
  ┌──────────────────────────────────┐
  │ ! ВНИМАНИЕ                      │
  │ Темп воды выход: 94 C           │
  │ Норма: 71–91 C                  │
  │ 12:34:56                        │
  └──────────────────────────────────┘
  ```

### Состояние "Критично" (critical) — полноэкранный overlay

Поверх экрана кабины — полупрозрачный затемнённый overlay с карточкой:

```
┌──────────────────────────────────────────────────────────────────┐
│ (затемнённый фон — данные кабины видны, но приглушены)           │
│                                                                  │
│     ┌──────────────────────────────────────────────┐             │
│     │                                              │             │
│     │  [!]  КРИТИЧНО                               │             │
│     │                                              │             │
│     │  Температура воды на выходе: 116 C           │             │
│     │  Предел: 115 C                               │             │
│     │                                              │             │
│     │  >> СНЯТЬ НАГРУЗКУ С ДИЗЕЛЯ                  │             │
│     │                                              │             │
│     │                          [  Принял  ]        │             │
│     │                                              │             │
│     └──────────────────────────────────────────────┘             │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

**Поведение:**
- Красный бордер / фон карточки, крупный текст
- Рекомендация действия: конкретная инструкция для машиниста
- Кнопка "Принял": overlay исчезает, алерт остаётся в списке со статусом "acknowledged"
- Звуковой сигнал: короткий зуммер при появлении (Web Audio API)
- Анимация: карточка мигает (border pulse) пока не нажато "Принял"

### Таблица алертов и рекомендаций

| Событие | Текст алерта | Рекомендация |
|---|---|---|
| Темп воды > 115 C | Температура воды: {value} C | СНЯТЬ НАГРУЗКУ С ДИЗЕЛЯ |
| Темп масла > 96 C | Температура масла: {value} C | СНЯТЬ НАГРУЗКУ С ДИЗЕЛЯ |
| Земля сил. цепей | Замыкание на землю в силовых цепях | НАГРУЗКА СНЯТА АВТОМАТИЧЕСКИ (ДГУ) |
| Земля вспом. цепей | Замыкание на землю во вспом. цепях | ПРОВЕРИТЬ ВСПОМОГАТЕЛЬНЫЕ СИСТЕМЫ |
| Расход воздуха вне нормы | Расход воздуха: {value} (норма 1500–3200) | СНЯТЬ НАГРУЗКУ С ДИЗЕЛЯ |
| Давление ГР < 7.0 | Давление ГР: {value} кгс/см2 | ПРОВЕРИТЬ ПНЕВМОСИСТЕМУ |

```
└──────────────────────────────────────────────────────────────────┘
```

### Отличия от экрана "Кабина" (экран 2)
- Кнопка "<- Назад к списку" в header
- Алерты имеют кнопку "Принял" и статус (active / acknowledged / resolved)
- Панель действий внизу: Replay, Экспорт, Отметить событие
- Данные обновляются live (WebSocket), как у машиниста

### Жизненный цикл алертов на этом экране
```
ACTIVE (новый)        — красный/жёлтый фон, мигает, кнопка "Принял"
  |
  v  (диспетчер нажал "Принял")
ACKNOWLEDGED          — приглушённый фон, не мигает, текст "принято"
  |
  v  (параметр вернулся в норму)
RESOLVED              — серый, перечёркнутый, уходит из списка через 60 сек
```

## Экран 6: Replay (`/dispatch/loco/:id/replay`)

**Назначение:** Перемотка и воспроизведение исторических данных за последние 5–15 минут.

### Layout

```
┌──────────────────────────────────────────────────────────────────┐
│ Header: [<<] РЕЖИМ REPLAY    TE33A-0231    [Вернуться к Live ->]│
├──────────────────────────────────────────────────────────────────┤
│                                                                  │
│  == Те же виджеты что на экране 5, но данные исторические ==     │
│  == Значения меняются синхронно с позицией на таймлайне    ==    │
│                                                                  │
│  ┌──────────────────────────────────────────────────────────┐   │
│  │ ТАЙМЛАЙН                                                 │   │
│  │                                                          │   │
│  │  12:20     12:25     12:30     12:35                     │   │
│  │  |-----!----|---------|----V----|                         │   │
│  │       ^алерт              ^текущая позиция               │   │
│  │                                                          │   │
│  │  [|<] [<<] [>/||] [>>] [>|]    Скорость: [1x] [2x] [5x]│   │
│  └──────────────────────────────────────────────────────────┘   │
│                                                                  │
└──────────────────────────────────────────────────────────────────┘
```

### Интерактивность
- Drag по таймлайну — перемотка к нужному моменту
- Кнопки управления: начало, назад, play/pause, вперёд, конец
- Выбор скорости воспроизведения: 1x / 2x / 5x
- Маркеры на таймлайне: алерты (красные/жёлтые метки) и отметки событий (pin-метки)
- Клик на маркер — прыжок к этому моменту
- Кнопка "Вернуться к Live" — возврат на экран 5 с live-данными

```
├──────────────────────────────────────────────────────────────────────┤
```

```
│ [ЭЛЕКТРИКА: тяга, генератор + земля toggles]                        │
```
